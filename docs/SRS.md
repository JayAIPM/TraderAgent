# 需求规格说明书（SRS）
## 待办事项垂直AI Agent

---

## 1. 项目概述

| 项目名称 | 待办事项垂直AI Agent |
|---------|---------------------|
| 产品定位 | 垂直固化架构AI Agent，专注文档待办全生命周期管理 |
| 核心约束 | 架构层锁死能力边界，仅支持4种待办操作，拒绝所有跨界任务 |

## 3. 数据模型

### 3.1 待办事项（Todo）

```javascript
{
  id: String,           // 唯一标识（UUID）
  title: String,         // 待办标题（必填）
  dueDate: Date,        // 截止时间（精确到时分秒）
  priority: String,     // 优先级：'high' | 'medium' | 'low'
  status: String,        // 完成状态：'pending' | 'completed'
  createdAt: Date        // 创建时间
}
```

### 3.2 字段规则

| 字段 | 规则 |
|------|------|
| id | 自动生成UUID |
| title | 自然语言提取，存储原始意图描述 |
| dueDate | 未指定默认当天09:00 |
| priority | 未指定或不确定时，向用户追问确认 |
| status | 新增默认 'pending' |
| createdAt | 自动记录当前时间 |

## 4. 核心功能规格

### 4.1 新增待办

| 规格项 | 详情 |
|--------|------|
| 输入示例 | "明天下午三点开会"、"提交项目报告"、"紧急处理线上故障" |
| 参数提取 | title（标题）、dueDate（时间）、priority（优先级） |
| 时间解析 | 支持自然语言："明天"、"后天"、"下周一"、"下午3点" |
| 优先级识别 | "紧急"/"重要"/"必须" → high；"尽快" → medium；无关键词 → 向用户追问 |
| 输出结果 | 展示待办ID、内容、时间、优先级、状态 |

### 4.2 查询待办

| 规格项 | 详情 |
|--------|------|
| 查询方式 | 全部查询 / 未完成查询 / 模糊搜索 |
| 模糊搜索 | 支持标题内容模糊匹配 |
| 时间范围 | 支持"本周"、"本月"等范围查询 |
| 排序规则 | 优先按截止时间排序 |
| 空数据处理 | 提示"暂无待办事项" |

### 4.3 删除待办

| 规格项 | 详情 |
|--------|------|
| 删除方式 | 根据ID或标题精准匹配删除 |
| 确认机制 | 前端弹出确认对话框，用户点击确认后执行 |
| 找不到处理 | 提示"未找到指定待办" |
| 输出结果 | 删除成功/失败原因 |

### 4.4 清空待办

| 规格项 | 详情 |
|--------|------|
| 用户输入 | "清空所有待办"、"删除全部" |
| 确认机制 | 前端弹出确认对话框，用户点击确认后执行 |
| 输出结果 | 提示"所有待办已清空" |

## 5. Agent执行流程

```
用户输入自然语言
    ↓
意图识别 → 是否为待办业务？
    ↓ 是                          ↓ 否
技能匹配                      返回拒绝信息：
    ↓                          "我仅支持待办事项管理相关操作，
参数提取                      无法为您提供其他服务。"
    ↓
业务执行（调用Service层）
    ↓
结果结构化返回
```

### 5.1 拒绝跨界请求规则

- 识别范围：闲聊、知识问答、代码编写、文案创作、数据计算、文件处理等
- 响应话术："我仅支持待办事项管理相关操作，无法为您提供其他服务。"
- 严格禁止：不做任何应答、不调用模型原生能力、不自主发挥

## 6. API接口规格

### 6.1 响应格式

```javascript
// 成功响应
{
  "code": 200,
  "msg": "操作成功",
  "data": { ... }
}

// 错误响应
{
  "code": 400,          // 业务错误码
  "msg": "错误描述",
  "data": null
}
```

### 6.2 错误码规范

| code | 含义 | 使用场景 |
|------|------|---------|
| 200 | 成功 | 操作成功 |
| 400 | 参数错误 | 无效的待办ID、参数校验失败 |
| 404 | 资源不存在 | 待办不存在 |
| 500 | 服务器错误 | 数据库异常、Agent调用失败 |

## 8. 异常处理

| 场景 | 处理方式 |
|------|---------|
| Ollama服务未启动 | 返回友好提示："AI服务暂不可用，请检查配置" |
| Ollama模型缺失 | 返回友好提示："AI模型未加载，请先拉取模型" |
| MongoDB连接失败 | 返回500错误，提示检查数据库配置 |
| 无效的待办ID | 返回400错误："无效的待办ID" |
| 删除不存在的待办 | 返回404错误："未找到指定待办" |
| 空数据库查询 | 返回空数组，提示"暂无待办事项" |
